概览

对外展示建议：讲清楚“角色 - 流程 - 数据来源 - 价值”。

模块

  • 移动端：上链信息录入（批次、种植、质检等）。
  • 小程序：扫码溯源查询（批次链路、状态与证明）。
  • Web：统计看板（种植 / 物流 / 销售多维）。

你的贡献

  • Vue3 + Vite + Pinia 基建与模块化拆分，提升可维护性。
  • 对接区块链接口、异常兜底与数据校验。
  • ECharts 看板联动、筛选与复用组件沉淀。

核心流程（建议配图）

把下面占位文字替换成你的截图或录屏帧（脱敏）。

流程 1：移动端上链录入
登录 → 选择基地/批次 → 录入种植/质检 → 提交 → 返回链上凭证（哈希/区块高度等，脱敏展示）
流程 2：小程序扫码溯源
扫码 → 拉取链上/链下数据 → 展示产地/物流/销售链路 → 异常提示（无数据/接口失败）
流程 3：Web 统计看板
筛选（时间/基地/品类）→ 图表联动 → 下钻详情 → 导出（如有）
工程化要点
路由懒加载、分包、资源优化、接口缓存/节流、统一错误处理。
